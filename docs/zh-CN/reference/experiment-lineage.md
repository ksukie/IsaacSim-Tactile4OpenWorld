# 实验版本谱系

仓库中存在两个相互独立的版本维度：

- Isaac Lab `2.1.1` 与 `2.3.2` 表示外部运行基线。
- OpenWorldTactile `V1` 至 `V6.2` 表示 2.1.1 触觉基准内部的研究阶段与探针。

`V6.2` 并不表示“比 Isaac Lab 2.3.2 更新”；两组数字描述的对象不同。

## 用户入口状态

| 阶段 | 推荐用途 | 状态 |
|---|---|---|
| V1 | 首次固定柔性膜 UIPC 运行与稠密 fxyz 输出 | 已记录的入门入口 |
| V5.7d–V5.7f | 构建后续估计器所需的坐标/形变契约 | 已记录的 V6.2 前置流程 |
| V5.7g | 后续阶段使用的冻结形变到力估计器 | 内部算法边界；有离线测试 |
| V5.9 | V6.2 复用的 TU 力场渲染辅助模块 | 实现依赖，不是首个用户入口 |
| V6.2 | Piper/PhysX/UIPC 闭环抓取 | 当前进阶场景 |
| 其他版本脚本 | 复现诊断或研究方法演进 | 研究历史；使用前检查 |

## 主要阶段

| 系列 | 主要研究问题 |
|---|---|
| V1 | 单张 UIPC 柔性膜能否生成稠密三通道形变触觉场？ |
| V2–V2.8 | 能否将力估计、相机观测膜面、标记点跟踪、纹理/印痕生成和样本打包拆分为可复用步骤？ |
| V3 文档 | 传感器挂载前需要怎样的验证套件？ |
| V4–V4.9c | 柔性膜、相机和力场能否随机器人传感器运动而不产生刚体运动伪影，并验证 USD 接触？ |
| 早期 V5 | 能否集成挂载接触与抓取/插孔场景？ |
| V5 `new` 1–6 | 能否分别定位挂载对齐、PhysX 抓取、接触间隙、视觉层、梯度/符号标定、压力/剪切代理和本构形变问题？ |
| V5.7a–V5.7g | 能否定义并验证柔性膜形变、连接/跟随、坐标契约、重复性和冻结估计器？ |
| V5.8–V5.9 | 抓取集成与局部 TU 触觉场能否复用冻结估计器？ |
| V6.0–V6.1b | 能否验证抓取/抬升/完整周期行为和失败诊断？ |
| V6.2 | 能否在保持触觉估计独立的同时，把原始 UIPC 接触反力反馈到 PhysX 独占的物体和夹爪？ |

原始阶段文档还讨论了潜在的数据质量与真实力标定阶段。规划阶段不等于已实现或已验证的发布功能。

## 依赖关系

V6.2 不是独立脚本：

```text
V5.7d + 五次 V5.7e 运行
          │
          ▼
     V5.7f 契约
          │
          ├── V5.7g 冻结估计器
          │          │
          │          ▼
          └──── V5.9 力场辅助模块
                         │
                         ▼
                       V6.2
```

只复制 `OpenWorldTactile_v6_2_grasp.py` 会缺少本地导入、资产和输入契约。

## 如何选择脚本

1. 新用户先运行 V1。
2. 复现当前抓取结果时，使用完整 V6.2 流程。
3. 研究算法问题时，选择文件名和保留笔记与问题最匹配的最小探针。
4. 执行历史阶段前检查导入、必需文件、默认值和验收逻辑。
5. 不要根据更大的版本号推断稳定性。

维护工具生成的[入口矩阵](../../internal/ENTRYPOINT_MATRIX.md)列出所有检测到的实验/工具脚本、直接运行信号、导入根和引用资产。它是清单，不是支持矩阵。
