# 当前最终实现

## 模拟和真实 SDK 的差距

真实 OpenWorldTactile SDK 需要的是相机采到的触觉 RGB 图：包含 baseline、胶体纹理、光照变化、法向压陷和切向位移。IsaacLab 里 Franka 夹爪本身只能通过 ContactSensor 得到接触点、接触力和总力，不能直接给出真实传感器的逐像素 RGB 或逐点压力分布。

当前做法是：用真实物理接触力作为门控，把接触点和总力投影到夹爪虚拟平面上，通过 COP + 最大熵方法生成 97 x 77 的 force_map，再模拟 OpenWorldTactile 的纹理、压陷和切向位移，渲染成 SDK 可输入的 RGB，最后用 OpenWorldTactile SDK 解算 FXYZ。

主脚本：

```bash
experiments/franka/current/cop_rgb_sdk.py
```

## 实现流程

```text
ContactSensor
-> contact_pos_w / force_matrix_w / net_forces_w
-> 接触点投影到手指局部 x/z 平面
-> total_force + COP
-> 最大熵 distribution
-> 97 x 77 force_map
-> force_map 归一化为 OpenWorldTactile pressure_map
-> 局部切向力生成 displacement_x/y
-> OpenWorldTactileRGBRenderer 生成带纹理 RGB
-> IsaacLabOpenWorldTactileBridge 输入 RGB
-> OpenWorldTactile SDK 输出 FXYZ
-> Isaac UI 显示左右 RGB + 左右 FXYZ
```

## Baseline

baseline 在仿真主循环前生成：

```python
zero_rgb = renderer.render_baseline()
```

内容：

```text
pressure = 0
displacement_x = 0
displacement_y = 0
固定 OpenWorldTactile 纹理
```

默认输入 SDK 5 次：

```text
openworldtactile_baseline_frames = 5
```

## 默认触觉平面

```text
virtual_pad_size = 0.038 x 0.048 m
virtual_pad_resolution = 0.0005 m
shape = 97 x 77
```

## 运行指令

```bash
env TERM=xterm ./isaaclab.sh -p experiments/franka/current/cop_rgb_sdk.py \
  --max_steps 0 \
  --show_rgb_maps \
  --force_preview_every 1 \
  --force_preview_scale 3 \
  --contact_force_threshold 0.05 \
  --point_force_threshold 0.0 \
  --rgb_force_full_scale 0.0 \
  --openworldtactile_shear_px_per_n 0.45 \
  --openworldtactile_shear_clip_px 18.0 \
  --openworldtactile_texture_strength 36 \
  --openworldtactile_texture_blur 3 \
  --openworldtactile_pressure_blur 5
```

## 默认显示

LEFT RGB    RIGHT RGB
